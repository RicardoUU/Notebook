# Claude 插件推荐

> 更新时间：2026-04-22
> 主题：适合 Claude Code / Claude 工作流的高频插件与技能推荐。

这份清单先按“我实际会优先装什么”来整理，不追求全，只保留对日常开发和工作流最有帮助的几类。

## 1. Skill Creator

**用途：** Skill 创建

适合场景：
- 想把重复流程沉淀成 skill
- 想把一套固定工作流变成可复用模板
- 想把自己的提示词、规则、资源整理成结构化能力

我会把它当成 Claude 的“能力打包工具”。
如果你已经开始频繁重复某类任务，比如日报整理、设计转代码、固定代码分析流程，就值得把它写成 skill。

## 2. Code Review

**用途：** 代码评审

适合场景：
- 提交前做一轮 review
- 检查潜在 bug、边界情况、代码异味
- 补一轮“如果我是 reviewer 会挑什么问题”的视角

它的价值不只是找错，更重要的是让 Claude 从“写代码的人”切到“挑代码的人”。
复杂任务里，这种角色切换很有必要。

## 3. Claude MD Management

**用途：** `CLAUDE.md` 生成 / 维护

适合场景：
- 新项目初始化 `CLAUDE.md`
- 老项目的 `CLAUDE.md` 太乱，需要整理
- 想把测试命令、代码风格、仓库约定、工作流规则沉淀下来

这是很实用的一类插件，因为 `CLAUDE.md` 写得好不好，直接影响 Claude 在项目里的稳定性。
我会把它看成“Claude 项目说明书管理器”。

## 4. Code Simplifier

**用途：** 代码优化

适合场景：
- 重构过于复杂的代码
- 降低嵌套和条件分支复杂度
- 把难读的实现改成更直接、更易维护的版本

这个插件比较适合在“代码已经能跑，但看起来别扭”的时候用。
不是修 bug，而是专门做简化和可读性提升。

## 5. Superpower

**用途：** 工程化插件，继承了 plan / execute / review 等流程

适合场景：
- 做复杂功能
- 修复杂 bug
- 需要先计划再执行
- 想把 review、TDD、debugging、并行 agent 等流程做完整

如果说前面几个插件更像单点能力，那 `Superpower` 更像一套完整工程化工作流。
它的价值不在“多一个命令”，而在于逼 Claude 按更稳的方式工作，而不是一上来就直接改代码。

这是我最推荐在复杂开发任务里用的一类插件。

## 6. Figma

**用途：** 设计还原

适合场景：
- 根据 Figma 设计稿实现页面
- 提取设计上下文、变量、组件信息
- 做设计到代码的还原
- 检查设计稿与实现之间的偏差

如果你经常做 UI、前端、设计还原，这类插件非常有价值。
尤其在 `Figma -> 代码` 这条链路里，它比“单纯看图写页面”稳定得多。
