# 每日 AI 编程资讯｜2026-03-27

> 日期：2026-03-27
> 结论：**今日无重要更新（数量偏少，但有 2 条值得开发者立刻留意的变动）**。

## 产品更新

- **GitHub Copilot 下线 Gemini 3 Pro（改推 Gemini 3.1 Pro）**  
  GitHub 已在 3 月 26 日将 **Gemini 3 Pro** 从 Copilot 的各类体验中弃用，覆盖 Copilot Chat、inline edits、ask / agent mode 以及 code completions。对团队来说，这不是“模型名字变化”这么简单——如果你们在企业策略、默认模型选择或内部文档里仍写着 Gemini 3 Pro，现在就该改成 **Gemini 3.1 Pro**，同时检查管理员侧模型策略是否已放开。  
  原始链接：<https://github.blog/changelog/2026-03-26-gemini-3-pro-deprecated>

- **GitHub Copilot coding agent 新增“代修 PR 冲突”能力**  
  现在可以直接在 Pull Request 评论里 `@copilot`，让它执行“合并 main 并解决冲突”这类操作。这个点对真实工作流很实用：它不只是生成 patch，而是在自己的云端环境里尝试改动、验证构建/测试，再回推结果，适合处理那些机械但耗注意力的冲突修复。  
  原始链接：<https://github.blog/changelog/2026-03-26-ask-copilot-to-resolve-merge-conflicts-on-pull-requests>

## 实战工具流

- **轻量观察：Google 把“给 coding agent 喂最新文档知识”这件事，做成了可复用 skill**  
  Google DeepMind 分享了一个面向 Gemini API 开发的 agent skill 实验：给模型补一层“最新 SDK / 文档入口 / 示例代码”的轻量技能后，`gemini-3.1-pro-preview` 在其评测里的成功率从 **28.2% 提升到 96.6%**。这条不是新产品发布，但对所有 AI coding workflow 都很有启发：如果你在团队里频繁踩 SDK 版本、废弃接口、过期示例的坑，**AGENTS.md / Skills / MCP 文档入口** 很可能比单纯换模型更立竿见影。  
  原始链接：<https://developers.googleblog.com/closing-the-knowledge-gap-with-agent-skills/>

## 今天值得试什么

1. **检查你的 Copilot 默认模型与团队文档**：把所有仍引用 `Gemini 3 Pro` 的地方改成 `Gemini 3.1 Pro`，并确认企业管理员策略已允许替代模型。  
2. **拿一个有冲突的 PR 试跑 `@copilot`**：例如评论 `@copilot Merge in main and resolve the conflicts`，观察它在你们仓库里的成功率、测试通过率与可审阅性。  
3. **给你常用的 AI coding 工具补“最新知识入口”**：哪怕先从最小版本开始——在仓库里补一份 `AGENTS.md`，或给 agent 接上官方文档 / SDK changelog / MCP 文档源，优先解决“模型会写旧接口”这个最常见问题。
