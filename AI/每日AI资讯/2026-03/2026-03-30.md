## 实战工具流

- **Amp CLI 过去 24 小时继续高频构建，说明 frontier coding agent CLI 还在快速试错。** `@sourcegraph/amp` 在 npm 上连续出现多次新构建，适合把它看作“值得试，但不适合现在就锁死版本”的工具：如果你在团队脚本或 CI 里接入这类 agent CLI，最好先固定版本并留回滚余地。  
  原始链接：<https://www.npmjs.com/package/@sourcegraph/amp> / <https://ampcode.com/>

- **今天更值得关注的是工作流方向，而不是单点新品。** 这几天公开变化最一致的主线，仍然是“把 agent 放进更真实的开发环境”——例如私有基础设施部署、Git worktree、多会话并行、自动化触发、JetBrains/终端集成。这些方向短期内比单次模型宣传更影响真实开发效率，今天虽然没有新的重大发布，但值得按这个方向检查你自己的工具链。  
  原始链接：<https://cursor.com/changelog> / <https://windsurf.com/changelog>

## 今天值得试什么

- **把 agent 和 Git worktree 绑定起来。** 给一个真实仓库开两个 worktree：一个跑修 bug agent，一个跑文档/测试 agent，观察并行会话是否比单线程更省上下文切换。
- **给不稳定的 agent CLI 全部加版本锁。** 如果你在脚本里用 `opencode`、`amp` 或其他快速迭代 CLI，今天就把版本固定，并补一条回滚说明，避免明天构建行为悄悄变化。
- **审一遍“哪些任务必须在私网里跑”。** 如果团队已经在试 cloud agent，把代码、构建产物、密钥、数据库访问分级，先明确哪些任务适合自托管/内网执行，再决定是否切到 self-hosted agent。
