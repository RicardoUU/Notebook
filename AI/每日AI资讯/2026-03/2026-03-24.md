# 每日 AI 编程资讯｜2026-03-24

日期：2026-03-24

今天整体看，过去 24 小时内**重大发布不算多**，但有几条对 AI 编程工作流影响比较直接的更新，尤其集中在 **Claude 的跨设备/电脑操作能力** 和 **Copilot 的模型可用范围扩展**。

## 产品更新

- **Claude Cowork / Claude Code 开始支持“直接操作你的 Mac”**  
  Anthropic 宣布在 Claude Cowork 与 Claude Code 中开放电脑操作能力研究预览：当缺少直连工具时，Claude 可以直接控制浏览器、鼠标、键盘和屏幕来完成任务。当前面向 Pro / Max 用户、仅支持 macOS，且对新应用访问仍要求显式授权。  
  原始链接：https://claude.com/blog/dispatch-and-computer-use

- **GitHub Copilot 上线 Gemini 3.1 Pro，并扩展到 JetBrains / Xcode / Eclipse**  
  GitHub 将 Gemini 3.1 Pro 以 public preview 形式接入 Copilot，覆盖 github.com、移动端、VS Code、Visual Studio，以及 JetBrains IDE、Xcode、Eclipse 的聊天模型选择器。对企业/团队来说，这意味着多模型策略开始更自然地进入非 VS Code 工作流。  
  原始链接：https://github.blog/changelog/2026-03-23-gemini-3-1-pro-is-now-available-in-jetbrains-ides-xcode-and-eclipse

- **GitHub 预告 AI 驱动的安全检测，补齐非主流语言与配置文件覆盖**  
  GitHub 宣布将在 GitHub Code Security 中引入 AI-powered detections，补充 CodeQL 对 Shell/Bash、Dockerfile、Terraform(HCL)、PHP 等生态的检测覆盖，并计划在 Q2 早期进入 public preview。对实际开发来说，这更像是“把 AI 安全审查前移到 PR 流程里”，而不只是单独的安全产品宣传。  
  原始链接：https://github.blog/security/application-security/github-expands-application-security-coverage-with-ai-powered-detections/

## 实战工具流

- **“手机派单 + 桌面执行”开始变成 Claude Code 的真实用法**  
  这次 Anthropic 的重点不只是 computer use 本身，而是和 Dispatch 组合：你可以在手机上把任务丢给 Claude，回到电脑时直接查看已完成结果。对开发者来说，适合交给它做低风险长链路任务，比如跑一轮测试、改 IDE 内文件、整理 PR 草稿，但前提是权限边界要收紧，别一上来就碰敏感项目。  
  原始链接：https://claude.com/blog/dispatch-and-computer-use

- **Claude “技能化补上下文”这条路，开始出现更成熟范式**  
  Simon Willison 用 Claude skill 给刚发布的 Starlette 1.0“补最新知识”，再让 Claude 直接生成符合新版本 API 的应用。这对所有版本迭代快、模型知识容易过期的库都很有启发：与其反复在 prompt 里解释 breaking changes，不如把新版本文档整理成可复用 skill / context 包。  
  原始链接：https://simonwillison.net/2026/Mar/22/starlette/

- **社区开始尝试把 Claude Code 能力拆成可脚本化 SDK / bridge**  
  `claude-code-sdk` 这个开源项目本身不是官方发布，但它代表了一个很实用的方向：把交互式 coding agent 能力抽成单文件 CLI、REPL 和 NDJSON bridge，方便挂进自动化流程、外部工具或自定义 agent 编排。现阶段更适合把它当成“工作流探索样本”而不是生产依赖。  
  原始链接：https://github.com/SeifBenayed/claude-code-sdk

## 今天值得试什么

- **试一次低风险的 Claude computer use 场景**：挑一个非敏感仓库，让它帮你跑测试、开浏览器查文档、改一两个文件，重点观察授权体验和失败重试成本。  
- **在 Copilot 里切到 Gemini 3.1 Pro 对比同一任务**：用同一段代码重构、测试生成或解释复杂报错，比较和默认模型在速度、代码风格、上下文保持上的差异。  
- **把“新版本说明”做成固定上下文包**：找一个你最近在升级的库，把 release notes + breaking changes 整成一份可复用文档，接下来让 agent 按这份上下文来改代码。
