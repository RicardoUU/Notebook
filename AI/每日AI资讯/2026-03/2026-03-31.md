# 每日 AI 编程资讯

日期：2026-03-31

## 产品更新

- **OpenHands 发布 1.6.0**  
  这是过去 24 小时里相对最值得记的一次正式版本更新。对日常使用最相关的是新增 `hooks`、`/clear`（保留当前 sandbox 和配置但重置对话历史），以及默认全局 skills 的开关；另外还修了会话持久化和导出重复事件的问题。  
  原始链接：https://github.com/OpenHands/OpenHands/releases/tag/1.6.0

- **今日大厂侧无特别重大的正式更新**  
  按过去 24 小时公开可核实信息看，Codex、Claude Code、Gemini、Cursor、Copilot、Windsurf、Devin、Warp、Zed 等主流 AI 编程产品没有出现足以进入日报头条的正式发布或权限策略大变更。今天更像是“开源工具持续打磨日”，而不是“旗舰产品改版日”。  
  参考链接：https://github.com/openai/codex/releases / https://github.com/continuedev/continue/releases / https://github.com/OpenHands/OpenHands/releases

## 实战工具流

- **OpenHands 的 `/clear` 很适合长任务“重置上下文、不重开环境”**  
  这类能力对 agent 工作流很实用：当一次任务聊太长、上下文变脏时，可以保留现有 sandbox / 配置，直接开一段干净对话继续做事，减少“重启环境 + 重新描述上下文”的摩擦。对调试、复现 bug、分阶段交付尤其有用。  
  原始链接：https://github.com/OpenHands/OpenHands/releases/tag/1.6.0

- **Continue 新版把配置和历史进一步往“多工作区可管理”方向推进**  
  虽然不是今天发布，但最近一版 `v1.3.38-vscode` 里两个点很实用：一是支持 `.continue/configs`，二是 session history 支持按 workspace directory 过滤。对多仓库、多客户、多项目并行的人来说，这能明显降低配置串线和历史污染。  
  原始链接：https://github.com/continuedev/continue/releases/tag/v1.3.38-vscode

- **OpenCode 在 24 小时内连续小版本修补，重点落在插件稳定性**  
  `v1.3.9` 修的是“无前导点路径的插件入口解析”，`v1.3.8` 则写明“无显著变化”。这类更新不适合当产品新闻看，但对已经把 OpenCode 接进本地工作流的人，意味着插件加载链路还在快速打磨，近期升级要留意自定义插件路径。  
  原始链接：https://github.com/anomalyco/opencode/releases/tag/v1.3.9

## 今天值得试什么

- 如果你在用 OpenHands，专门试一次：长对话做一半后执行 `/clear`，观察“上下文清空但运行时保留”是否真的比重开 session 更顺手。
- 如果你在 VS Code 里用 Continue，把项目级配置挪到 `.continue/configs`，并验证多仓库切换时是否更容易控住模型、命令和规则。
- 如果你在折腾 OpenCode 插件，检查自定义插件入口路径写法；升级到 `v1.3.9` 后做一次冷启动验证，重点看插件是否能稳定加载。
