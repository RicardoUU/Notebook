## 产品更新

### 1. Codex CLI 0.118.0 发布：补上更实用的 `exec` 管道流、动态鉴权与更严的沙箱网络控制
- OpenAI 在 **3 月 31 日 17:02 UTC** 发布 `openai/codex` `0.118.0`。对真实工作流最有用的点有三个：`codex exec` 现在支持“命令行 prompt + stdin 管道”混合输入；自定义模型提供方支持动态刷新短期 bearer token；Windows 沙箱可用操作系统级出口规则强制走代理，不再只靠环境变量。
- 这版也修了一些影响日常稳定性的坑：首次创建项目级 `.codex` 文件时的审批保护缺口、Linux `bwrap` 发现问题、MCP 启动告警缺失、Windows `apply_patch` 可靠性等。
- 原始链接：https://github.com/openai/codex/releases/tag/rust-v0.118.0
- 发布时间：**发布于今日凌晨前（2026-03-31 17:02 UTC）**

### 2. Claude Code 2.1.89 发布：补强 hook / MCP / 长会话稳定性
- Anthropic 在 **4 月 1 日 01:07 UTC** 发布 `claude-code` `v2.1.89`。这次更像一版“把高频卡点都抹平”的工程化更新：`PreToolUse` hook 新增 `defer` 决策，适合 headless 会话先暂停、再用 `-p --resume` 继续；`PermissionDenied` hook 能在 auto mode 拒绝后触发二次重试逻辑；`MCP_CONNECTION_NONBLOCKING=true` 则能让 `-p` 模式跳过 MCP 连接等待。
- 另外还有一串很值钱的稳定性修复：多 schema `StructuredOutput` 失败率、超长会话内存泄漏、LSP crash 后僵死、CJK/emoji 历史丢失、subagent token 统计不准、Windows CRLF / Markdown 空格处理等。
- 原始链接：https://github.com/anthropics/claude-code/releases/tag/v2.1.89
- 发布时间：**发布于今日（2026-04-01 01:07 UTC）**

### 3. OpenCode 连续滚动更新到 1.3.13：最近一波重点是 Anthropic/Vertex 缓存与插件安装安全
- `anomalyco/opencode` 在 **4 月 1 日 00:44 UTC** 发布 `v1.3.13`，但真正对工作流有意义的改动集中在过去几小时内的 `v1.3.12`、`v1.3.11`：为 Google Vertex Anthropic 打开 prompt caching 与 cache token tracking，修复 AI SDK v6 迁移后 Azure provider 参数透传，改善 TUI 外部命令输出，以及收紧插件安装行为（固定插件版本、阻止包安装脚本运行）。
- 如果你把 OpenCode 当成多 provider / 多插件的 agent CLI，这一波明显是在继续补“可运维性”和“更少脏边角”。
- 原始链接：https://github.com/anomalyco/opencode/releases
- 发布时间：**最新发布于今日（2026-04-01 00:44 UTC）**

## 实战工具流

### 4. Cloudflare Wrangler Workflows 支持 `--local`：本地把工作流全生命周期跑通了
- Cloudflare Changelog 在 **4 月 1 日** 更新，宣布所有 `wrangler workflows` 命令都支持 `--local`，可以直连本地 `wrangler dev` 会话，而不是只打生产 API。
- 对 agent / 自动化工作流开发者，这很实用：触发、列实例、暂停、恢复、重启、终止、发事件，现在都能本地演练；做 AI agent 编排或异步任务流时，调试门槛明显更低。
- 原始链接：https://developers.cloudflare.com/changelog/
- 发布时间：**更新于今日（2026-04-01）**

### 5. Vercel AI SDK 给 Gemini 3 补上“内建工具 + function calling”组合能力
- `vercel/ai` 在 **4 月 1 日 00:41 UTC** 发布 `@ai-sdk/google@3.0.55`，同时 beta 线 `@ai-sdk/google@4.0.0-beta.19` 也在 **00:16 UTC** 带上同一能力：Gemini 3 现在支持把 provider 内建工具与函数调用一起用。
- 对基于 AI SDK 搭 agent 的人，这个点比名字看起来更重要：以前常常要在“模型原生工具”和“自定义函数工具”之间做割裂的封装，现在组合空间更大，工具编排会更顺手。
- 原始链接：https://github.com/vercel/ai/releases
- 发布时间：**发布于今日（2026-04-01 00:16–00:41 UTC）**
